Requirment Identifications and Analysis

Whether it is a new system or upgrading an existing system, Westech provides the expertise to identify, document, and verify the requirements with the organization's stakeholders and everyday users. For existing systems, as a new customer requirement is requested, Westech follows a Best Practice process:

  1. Validate the request - does existing functionality provide the same result?
  2. Clarify the requirement – document the requirement details with the requesting party
  3. Conﬁrm the request across the user community – ensure the new function will not adversely affect the application(s)
  4. Guide the requirement through any internal review boards to receive approval for design and development
  5. Obtain management sign-off for implementation of the requirement
  6. Interface with the architects and engineers to translate the customer requirement into a system requirement

When a new application or system is the desired result, Westech is proﬁcient in writing Agile user stories as well as using traditional methods. The choice of methods is up to the customer – one-on- one interviews, group interviews or facilitated sessions to obtain the requirements. The number and type of sessions will depend on:

  1. The nature of the application/system
  2. Size of the user community
  3. Geography of the user community, i.e., headquarters vs. remote ofﬁces, local vs. global, etc
  4. Interfaces with existing systems

At Westech, our consultants can assist your IT staff to identify and document new requirements or just review those already gathered for completeness. Due to our industry experience in multiple phases of the development process, Westech can drive the requirements from identiﬁcation through to deployed system functionality.
